# Change History

This log tracks major decisions and milestones as the scaffold application evolves.

| Date | Change | Notes |

| 2026-03-28 | Audit trail expansion and export audit logging (2026.3.16) | Massively expanded `_DEFAULT_AUTO_AUDIT_CONFIG` in `scaffold/core/audit.py` to cover all application models by default — no manual `AUDIT_LOG_MODEL_EVENTS` configuration required. Added identity sub-models (AADGroupMapping, MFASetting excluding sensitive credentials, PasskeyCredential excluding raw key material), expanded User fields (`username`, `azure_oid`, `aad_upn`), and added full coverage for BIA (BiaTier, AuthenticationMethod, InformationLabel, ContextScope, Component, ComponentEnvironment, Consequences, and more), CSA, DPIA, Risk, Incident, Maturity, and Threat models. Added `log_event` audit calls to every export endpoint across all modules (BIA: HTML/CSV/SQL/data-inventory/auth-overview/consequences/availability/dependencies; CSA: HTML/PDF; Incident: HTML/PDF; Maturity: HTML/PDF; Threat: CSV/HTML/PDF). Fixed nginx upstream resolution: replaced the static `upstream app_backend` block with Docker's internal DNS resolver (`127.0.0.11`) and a dynamic `$upstream` variable so container hostnames are re-resolved after rebuilds without restarting the gateway. Simplified `AUDIT_LOG_MODEL_EVENTS` documentation in `env.production.example` — leaving it empty now enables built-in defaults covering all models. Added per-component availability and consequences views in the BIA module with dedicated forms, help text, and updated translations. |
| 2026-03-25 | DPIA assessment detail page restyled | Rewrote `assessment_detail.html` to match the application's Tailwind conventions. Removed the redundant double-wrapping container div (base layout already provides the `max-w-5xl` container). Replaced Bootstrap-style `flex flex-wrap -mx-3 g-3` form/grid patterns with native Tailwind `grid grid-cols-N gap-4`. Updated all four cards (Details, Questions, Risks, Measures) and their inline edit modals to use `bg-[var(--color-surface)]` thead, `px-4 py-3` cell padding, `divide-y divide-[var(--color-border)]` tbody separators, and `[&_tbody_tr]:hover:bg-primary-500/5` row hover (matching BIA/threat modules). Removed duplicate CSS class declarations. Replaced Bootstrap-only classes (`d-inline`, `modal-lg`, `modal-dialog-centered`, `btn-close-white`) with Tailwind equivalents. Restyled the sidebar `<dl>` from a flex-gutter layout to `grid grid-cols-[5fr_7fr]`. Added `font-semibold` to all card header labels. |
| 2026-03-23 | Threat module: Bootstrap → Tailwind migration (2026.3.14) | Removed all Bootstrap utility classes and JS components from all 13 threat module templates. Replaced layout helpers (`d-flex`, `flex-column`, `fw-*`, `fs-*`, `ms-*`, `me-*`, `text-uppercase`, `text-muted`, `small`, `badge bg-*`, `row g-3`/`col-*`, `table table-sm`, `form-control`, `form-select`, `form-check`) with Tailwind equivalents. Replaced Bootstrap CSS variables (`--bs-border-color-translucent`, `--bs-secondary-color`, `--bs-tertiary-bg`, `--bs-body-color`) with app theme variables (`--color-border`, `--color-muted`, `--color-surface-hover`, `--color-text`).
